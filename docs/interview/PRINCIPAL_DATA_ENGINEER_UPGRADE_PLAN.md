# Principal Data Engineer Upgrade Plan

## Why upgrade the original CAPSDAC project?

The original public repository already demonstrates enrollment forecasting and stakeholder reporting. For a Principal Data Engineer role, the project must also show platform thinking: data contracts, governance, orchestration, monitoring, cloud architecture, CI/CD, privacy controls, and operational readiness.

## Upgrade themes

1. **From notebook workflow to data product**: reusable source modules, tests, and scripts.
2. **From model output to governed forecast service**: data contracts, quality checks, lineage-ready outputs.
3. **From reporting demo to executive decision platform**: statewide, vendor, and site-level outputs with decision implications.
4. **From local project to cloud architecture**: AWS lakehouse layers, Glue, Step Functions, SageMaker, Athena, IAM, KMS, monitoring.

## Resume framing

Built a governed education forecasting data platform for CAPSDAC-style enrollment planning, including data contracts, lakehouse-ready architecture, time-series feature engineering, recursive machine learning forecasts, contribution analysis, executive reporting, and cloud productionization design.
